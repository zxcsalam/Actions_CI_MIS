# CI report (GitHub Actions)

## 1) Что такое CI (своими словами)
CI (Continuous Integration) — это практика автоматической проверки кода при каждом изменении. Работает как "автоматический вахтер": как только разработчик отправляет код (git push), сервер (GitHub Actions) запускает скрипт, который собирает проект и гоняет тесты.

В ИБ-проектах это критически важно для реализации концепции DevSecOps. Это позволяет находить уязвимости и баги на ранней стадии ("shift left"), гарантируя, что в ветку main не попадет код, который ломает логику безопасности или функционал приложения.

## 2) Что проверяет ваш workflow
Мой файл `.github/workflows/tests.yml` выполняет следующие шаги:
1.  Checkout: клонирует репозиторий на виртуальную машину.
2.  Setup Python: устанавливает версию Python 3.11.
3.  Install dependencies: обновляет pip и устанавливает библиотеку `pytest`.
4.  Run tests: запускает команду `python -m pytest -q` для проверки кода.
